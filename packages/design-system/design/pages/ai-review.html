2SQL · LEFT JOIN12.4s
-- orders: 5 строк, 2 без customer_id
SELECT COUNT(*) FROM orders o
LEFT JOIN customers c ON o.customer_id = c.id
WHERE c.country = 'RU';
ваш ответ: вернёт все 5 строк
верно: только строки с клиентом из RU

Условие WHERE c.country = 'RU' отсеивает строки, где после LEFT JOIN c равен NULL — внешнее соединение фактически становится внутренним. Перенесите условие в ON, чтобы сохранить заказы без клиента.

5Конкурентностьистёк
// два потока инкрементируют counter
counter = counter + 1;
ваш ответ: всегда +2
верно: возможна гонка, итог +1

Операция counter + 1 не атомарна: чтение и запись разнесены. Если потоки прочитают одно значение до записи, один инкремент потеряется. Нужен атомарный счётчик или мьютекс.
