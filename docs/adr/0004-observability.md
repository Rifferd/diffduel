# ADR-0004: Observability — OTel везде, полный стек в dev, лёгкий в проде

- **Статус:** принято
- **Дата:** 2026-06-14
- **Связано с:** ADR-0001 (прод-VPS 2 vCPU / 4 GB)

## Контекст

ТЗ §3.12 требует три сигнала (logs/metrics/traces), trace context propagation сквозь API→Kafka→worker, RED-метрики и продуктовые (`duel_match_time_seconds`, `answer_check_latency`, `ws_connections`). Но прод-VPS — 4 GB, и полный стек Grafana+Tempo+Loki+Prometheus там не помещается рядом с приложением.

## Решение

**Инструментирование — везде, экспорт — по флагу.** Все три сервиса (api/realtime/workers) инструментированы OpenTelemetry (traces + metrics). Экспорт включается переменной `OTEL_EXPORTER_OTLP_ENDPOINT`: если она пуста — OTel не инициализируется (no-op провайдеры, нулевой оверхед). Trace context (`traceparent`) инжектится в заголовки Kafka-сообщений в продюсере и извлекается в воркере — сквозной трейс «ответ игрока → дуэль завершена → карточка».

**Dev:** `infra/compose/compose.observability.yml` (оверлей) поднимает otel-collector → Tempo (traces) / Loki (logs) / Prometheus (metrics) → Grafana (анонимный вход, провижн датасорсов и дашборда DiffDuel как код). Сервисы на host экспортируют на `localhost:4317`. Это витрина для собеседования: открыл Grafana → показал RED + трейс сквозь сервисы.

**Прод:** observability-сервисы НЕ запускаются (память). Включены: **Sentry** (`SENTRY_DSN`) для ошибок + structured JSON-логи (уже есть). `OTEL_EXPORTER_OTLP_ENDPOINT` пуст → OTel no-op. При необходимости трейсинга в проде позже — экспорт на внешний управляемый OTLP-приёмник (Grafana Cloud free tier) одной переменной, без локального стека.

## Последствия

- (+) Полная observability-история для портфолио в dev; прод не рискует памятью.
- (+) Включение трейсинга в проде — одна env-переменная (внешний коллектор), код готов.
- (−) В проде сейчас нет распределённого трейсинга — только Sentry + логи. Для текущей стадии достаточно; при росте — внешний OTLP.
- Dev-стек тяжёлый (5 контейнеров) — поднимается отдельным оверлеем по требованию, не в дефолтном `up`.
