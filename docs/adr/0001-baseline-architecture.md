# ADR-0001: Базовая архитектура и прод-профиль на стартовом VPS

- **Статус:** принято
- **Дата:** 2026-06-12
- **Контекст:** продуктовое ТЗ `base_/TZ_DiffDuel.md` (§3–§4, §12)

## Контекст

ТЗ фиксирует стек (FastAPI + NestJS + PG + Redis + Kafka + MinIO + observability) и монорепо-структуру. Открытым оставался вопрос продакшен-площадки: исходный VPS (1 vCPU / 2 GB) занят тремя посторонними проектами за системным nginx — полный стек туда не помещается, а порты 80/443 заняты, что исключает Traefik из ТЗ.

Профиль нагрузки на старте: ≤ 200 одновременных пользователей, рост ожидается через несколько месяцев.

## Варианты

1. **Деплой на существующий VPS, облегчённый профиль** — без Kafka и observability-стека, события через Redis Streams. Дёшево (0 ₽), но риск деградации чужих проектов (своп уже используется) и отступление от ТЗ.
2. **Выделенный VPS 4 vCPU / 8 GB** — полный стек без компромиссов, ~1000–1500 ₽/мес. Переплата при текущей нагрузке.
3. **Выделенный VPS 2 vCPU / 4 GB, полный стек в ужатой конфигурации** — ~400–700 ₽/мес. Redpanda в однопоточном режиме (`--smp 1 --memory 400M --reserve-memory 0M`), observability с retention 3–7 дней, memory-limits на каждый контейнер. Расчётное потребление ~2.8–3 GB.

## Решение

Вариант 3: **выделенный VPS 2 vCPU / 4 GB** только под DiffDuel. Существующий сервер не используется вовсе. На своём сервере возможен Traefik (TLS, sticky sessions для WS) ровно по ТЗ. Все компоненты получают явные лимиты памяти в compose; вертикальный апгрейд тарифа — штатный путь масштабирования при росте.

Архитектурные решения ТЗ принимаются без изменений: раздельные Core API (stateless REST, Python) и Realtime (stateful WS, TypeScript), Kafka-протокол (Redpanda), MinIO с presigned URL, события с версией в конверте.

## Последствия

- (+) Существующие проекты вне зоны риска; полный стек ТЗ в проде — честная история для собеседований.
- (+) Traefik + Let's Encrypt по ТЗ, без компромиссов с чужим nginx.
- (−) Платная площадка (~500 ₽/мес).
- (−) Памяти впритык: обязательны memory-limits, мониторинг RSS и алерт на swap; observability-retention короткий.
- Деплой блокируется покупкой VPS — до этого вся разработка и проверка идут локально в dev-compose.
