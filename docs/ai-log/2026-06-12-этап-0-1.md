# AI-лог: 2026-06-12 — фундамент и Core API (этапы 0–1)

## Процесс

Оркестратор — Claude Fable 5, рутина — два параллельных Opus-субагента с непересекающимися файловыми зонами:

1. **Инфраструктура** (`infra/`, CI, Makefile, `.env.example`) — compose-стек PG/Redis/Redpanda/MinIO с healthchecks, идемпотентный minio-init, workflow api-ci.
2. **Core API** (`apps/api/`) — скелет FastAPI, 13 таблиц ТЗ §5 (answers партиционирована), Alembic, auth с ротацией refresh и reuse detection, rate limiting на Lua.

Контракт между агентами — `docs/specs/conventions.md`, написан до старта (порты, env-имена, слои, требования безопасности).

## Что AI сделал хорошо сам

- Поймал реальный баг тестом: отзыв family при reuse уезжал в rollback вместе с 401 → явный commit перед raise (`_revoke_family_durably`).
- `_DUMMY_HASH` против timing-оракула при логине несуществующего пользователя.
- PK `(id, submitted_at)` у партиционированной `answers` — требование PG, в спеке не оговаривалось.

## Что пришлось править на ревью (оркестратор)

1. **Ротация refresh была неатомарна**: два параллельных `/auth/refresh` одним токеном оба проходили проверку `rotated_at is None` и получали по новой паре токенов — reuse detection молчал. Исправлено атомарным `UPDATE … WHERE rotated_at IS NULL RETURNING id`; проигравший трактуется как reuse → отзыв family. Покрыто тестом.
2. **Rate limiting доверял `X-Forwarded-For` безусловно** — лимит обходился одним поддельным заголовком. Теперь XFF учитывается только при `TRUST_PROXY=true` (за нашим Traefik) и берётся последний элемент (его дописывает прокси, клиент подделать не может).
3. **Прод мог стартовать с dev-секретами** — добавлен fail-fast валидатор Settings: в `APP_ENV=prod` отклоняются dev-дефолты JWT_SECRET/INTERNAL_API_TOKEN и http-origin'ы в CORS. Покрыто тестами.
4. Точка входа uvicorn в Makefile (`app.main` → `src.main`) — агенты работали параллельно, инфраструктурщик честно пометил это допущение в отчёте.
