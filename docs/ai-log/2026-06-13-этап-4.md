# AI-лог: 2026-06-13 — лидерборды, share-карточки, админка, деплой (этап 4)

## Процесс

Спека `docs/specs/leaderboards-admin.md` (разделы A–E) — контракт. Три параллельных Opus-субагента:
- **API** (apps/api): лидерборды на Redis ZSET, публичные профили, админка с RBAC, feature flags.
- **Worker** (apps/workers с нуля): image-gen консьюмер `duels.finished` → рендер share-карточки → MinIO.
- **React-админка** (apps/admin с нуля): нарезка admin-дизайна, TanStack Table/Query, auth-гейт по роли.

Деплой-артефакты (раздел E) оркестратор делал сам — это интеграционная работа поверх всех сервисов.

## Что субагенты сделали хорошо сами

- API: `lb:global` через `ZADD GT` (score только растёт), отрицательное кэширование флагов (анти-стампида), скрытие забаненных из топа, EXPLAIN профиля в ADR-0002 (BitmapOr по двум индексам, ~0.23мс).
- Worker сам нашёл баг сравнения победителя по идентичности объекта (визуальный смоук выявил инверсию цветов) → заменил на `Literal["left","right"]`.

## Координация контрактов (оркестратор)

1. **Worker зависел от двух internal-эндпоинтов API**, которых не было в скоупе API-агента (`GET /internal/duels/{id}/card`, `POST .../share-card`). Оркестратор дописал их сам: метод `set_share_card_key` с идемпотентной записью (`UPDATE ... WHERE share_card_key IS NULL`), отдача ников игроков, +4 теста.
2. **Не было способа создать первого админа** (агент админки завёл его руками в БД для смоука). Для продакшена добавлен CLI `python -m src.create_admin` — пароль из env/getpass (не в аргументах→не в истории shell), идемпотентное повышение роли. Поймал и починил `Event loop is closed` (двойной `asyncio.run`).

## Живая верификация (не только тесты)

- **Share-карточка end-to-end:** реальная дуэль через internal API → событие в Redpanda → worker отрендерил → MinIO → ключ записан в дуэль → карточка раздана публично (HTTP 200, валидный PNG 1200×630). Визуально проверена: VS-сплит, счёт 5:0, дельты +16/−16, водяной знак diffduel.com.
- **Лидерборд и профиль:** на живых данных — ранги с обогащением никами (без N+1), агрегаты профиля (винрейт, streak).
- **Прод-образы:** собраны и запущены локально — api `/healthz` 200, web SPA history-fallback 200. Деплой не упрётся в кривой Dockerfile.

## Деплой (подготовлено, применение ждёт VPS)

`docker-compose.prod.yml` (memory-limits ~3.3 ГБ из 4, Traefik v3 + Let's Encrypt, sticky WS), Dockerfile'ы всех 5 сервисов, `.env.prod.example`, `deploy.yml` (build→push GHCR→SSH→миграции→smoke→откат; безопасно no-op без секретов). Решения — в ADR-0003.

API: ruff + mypy strict + 84 теста. Worker: ruff + mypy + 23 теста. Admin: eslint + tsc + 9 тестов + build. Прод-compose валиден, образы собираются. CI-workflow на все четыре бэкенд/фронт-приложения.
