# AI-лог: 2026-06-13 — соло-режим и фундамент SPA (этап 2)

## Процесс

Два параллельных Opus-субагента: соло-режим API (тренировки, проверка ответов, сид-контент, presigned-аватары) и фундамент Vue SPA (workspace, контракты, auth, нарезка дизайн-кода). Архитектурные решения зафиксированы оркестратором до старта: quiz-формат MVP с проверкой только на сервере, идемпотентный сид с фиксированными UUID, access-токен только в памяти SPA, single-flight refresh.

## Инцидент: смерть субагента

API-агент упёрся в лимит сессии на 71-м tool-вызове — код написан (домен tasks, checker, S3-клиент, аватарный флоу, 149/150 задач сида), но без тестов и с непрогнанными линтерами. Оркестратор доделал руками: 50-я задача сида, type-stubs aiobotocore, недостающие тесты (10 шт.: утечка эталона, ответы, presign/confirm, идемпотентность сида, happy-path против MinIO с graceful skip для CI).

**Вывод для процесса:** длинные задачи с генерацией контента (150 задач!) надо резать на два прогона — контент отдельно, код отдельно.

## Ревью оркестратора

- Код агента качественный: checker с диспетчеризацией по типу, отдельная публичная схема без `answer`/`explanation`, проверка принадлежности ключа аватара (`{user_id}/`), Content-Type зашит в подпись presigned PUT.
- Найдено и исправлено: дыра прод-гарда — S3_SECRET_KEY с dev-дефолтом прошёл бы в прод (добавлен в fail-fast валидатор + тест).
- SPA-агент отработал чисто: lint/vue-tsc/9 тестов/build перепроверены оркестратором, дизайн-система не тронута (контракт соблюдён), localStorage не используется.

## Валидация

API: ruff + mypy strict (51 файл) + 42 теста зелёные, сид идемпотентен (150 задач, 3 темы). Web: eslint (max-warnings 0), vue-tsc strict, vitest 9/9, vite build. Добавлен web-ci workflow.
