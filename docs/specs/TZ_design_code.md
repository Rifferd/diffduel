# ТЗ: Дизайн-код и полная верстка DiffDuel (web + mobile-web)

> Источники: `base_/TZ_DiffDuel.md` (продуктовое ТЗ, §13 — дизайн-система), `base_/design_web.html` (десктоп-макет), `base_/design_mobile.html` (4 мобильных экрана).
> Статус мобильной версии: **временно — адаптивное веб-приложение** (одна кодовая база, одна вёрстка; мобильные макеты задают поведение на узких вьюпортах). Capacitor/нативная оболочка — вне рамок этого ТЗ.

## 0. Цель и результат

Создать **дизайн-код** (формализованную дизайн-систему в CSS) и **статическую верстку всех страниц продукта** — web и mobile — как единый адаптивный набор HTML/CSS, готовый к последующей нарезке на Vue-компоненты (SPA) и React-компоненты (админка).

Результат (Definition of Done):

1. `packages/ui-tokens/tokens.css` — единый источник всех токенов, ноль захардкоженных цветов/радиусов/теней в вёрстке страниц.
2. `design/components/` — стайлгайд: каждый компонент со всеми состояниями на одной странице-витрине.
3. `design/pages/` — все страницы из §4 этого ТЗ, свёрстанные адаптивно (320–1440 px без горизонтального скролла), плюс ключевые состояния (пустое/загрузка/ошибка).
4. `design/index.html` — оглавление-превью со ссылками на все страницы и стайлгайд.
5. Соответствие чек-листу качества (§8).

## 1. Дизайн-направление (неизменяемое ядро)

Метафора: **дуэль = git diff**. Светлый «code-review» интерфейс; тёмные панели (`--arena`) — только зоны кода и live-дуэли.

Семантика цвета — жёсткая, декоративное использование запрещено:

- Зелёный (`--plus`) = вы / победа / верный ответ / рост.
- Красный (`--minus`) = соперник / поражение / ошибка / падение.
- Оранжевый (`--timer`) = только таймер и срочность.

Сигнатурные приёмы (обязательны к воспроизведению):

- **Диагональный VS-сплит** — `skewX(-12deg)` плашки и жёсткие градиенты `linear-gradient(102deg, … 49.6%, … 50.4%)` (hero, шапка дуэли, share-карточка, founder-блок, логотип-бейдж `.vs`).
- **Diff-строки** — статусы как `+24 Elo` / `−18 Elo`, маркеры списков `+`, лог «как проходит дуэль» с жёлобом-гаттером.
- **Все числа** (Эло, таймеры, счёт, время) — только JetBrains Mono + `font-variant-numeric: tabular-nums` (цифры не прыгают при тике).
- Подчёркивание-маркер в заголовках: `box-shadow: inset 0 -14px var(--plus-bg)`.
- Eyebrow-метки `// текст` — mono, uppercase, letter-spacing .12–.14em.

## 4. Полный реестр страниц

Каждая страница — **один адаптивный HTML** (не отдельные web/mobile файлы). Брейкпоинты:

- `≥1120px` — desktop (контейнер `.wrap` max-width 1120, padding 0 24px);
- `880px` — планшет: все multi-column grid → 1 колонка (как в макете);
- `≤640px` — mobile: вёрстка по мобильным макетам — таббар вместо верхней навигации (на app-страницах), full-width кнопки, стековые списки вместо таблиц.

Приоритет = порядку. Колонка «Реф.» — что брать за образец.

## 5. Адаптивность и mobile-web

- Подход: **desktop-first вёрстка с мобильной перестройкой** (макеты заданы для обеих крайних точек; 880px — переходная).
- На `≤640px` для app-страниц (блок B): верхняя nav скрывается, появляется фиксированный `.tabbar`; контент получает `padding-bottom` под таббар; safe-area через `env(safe-area-inset-bottom)`.
- Экран дуэли B3: на всех вьюпортах full-height (`100dvh`), без прокрутки контента задачи на mobile (код — внутренний overflow).
- Таблицы (`.board`, админка) на mobile превращаются в стековые строки `.lb` — не сжимать таблицу.
- Не использовать hover как единственный носитель информации (touch); активные состояния — `:active` + видимый focus.
- PWA-минимум в рамках вёрстки: `manifest.webmanifest` (имя, иконки, `theme_color: #161B22`, `background_color: #EEF1F5`), мета `viewport-fit=cover`, офлайн-страница A8. Service worker — вне рамок (этап SPA).

## 6. Доступность и движение

- Контраст AA для всех пар (проверить особенно: `--arena-soft` на `--arena-2`, `--timer` на арене, белый на `--plus`).
- Видимый фокус: `--focus-ring` на всех интерактивных элементах; навигация с клавиатуры по дуэли (варианты ответа — radio-семантика).
- `prefers-reduced-motion: reduce` → отключить: анимацию таймера `tick`, пульсации, transition transform; таймер остаётся (меняется числом).
- Семантика: лендинг — `header/main/section/footer`, заголовочная иерархия без пропусков; таблицы — `<table>` с `<th scope>`; таймер — `role="timer"`, фид дуэли — `aria-live="polite"`; счёт и Эло — `aria-label` с человекочитаемым текстом.
- `lang="ru"`, изображения и SVG-иконки с `aria-hidden`/`title` по роли.
