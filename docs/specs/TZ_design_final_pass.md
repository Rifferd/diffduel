# ТЗ: Финальный проход дизайна DiffDuel (этапы 4–6 + фиксы аудита)

> Один запрос — полное закрытие дизайн-фазы. После приёмки этого прохода начинается основная разработка (SPA/бэкенд), и дизайн-код становится read-only контрактом.
> Мастер-ТЗ: `docs/specs/TZ_design_code.md` (далее «§N» — его разделы). Рабочая папка: `DiffDuel Design System/`.
> Вариативность больше не нужна: по всем зонам без макетов принимай одно сильное решение в рамках ядра §1 — раунда согласования вариантов не будет.

После этого — заменить все вхождения этих hex в `components-base.css` / `components-domain.css` на токены; `#000`-бордеры арена-карточек (`components-domain.css:166`, `landing.html`, `index.html .tile--arena`) → `var(--frame)`.

### 1.2 Инлайн-цвета → модификаторы

Добавить в `components-domain.css`: `.share__d--plus { color: var(--plus-bright) }`, `.share__d--minus { color: var(--minus-bright) }`, `.f-card__res--win`. Убрать все 9 вхождений `style="color:var(--…)"` из `landing.html`, `duel-result.html`, `components-domain.html`. Также убрать `style="font-size:24px"` в `duel-result.html` (модификатор `.share--sm` или аналог).

### 1.3 Доступность (блокирует приёмку)

- `duel.html`: `role="timer"` на `.duel-timer__num`; `aria-live="polite"` на `.feed`; `role="radiogroup"` на `.opts` + `role="radio"`/`aria-checked` на `.opt`; обернуть арену в `<main>`, добавить visually-hidden `<h1>`.
- `<th scope="col">` во всех таблицах (`landing.html`, `leaderboard.html`, `components-domain.html`).
- `<h1>` (хотя бы `.visually-hidden`, класс добавить в `base.css`) на `app-home.html`, `duel-result.html`.
- `aria-hidden="true"` на декоративных SVG таббара везде.
- В демо модалки стайлгайда: `role="dialog"`, `aria-modal="true"`.

### 1.4 Вёрстка и поведение

- **JS на лендинге запрещён**: мобильное меню `toggleMenu()` переделать на CSS-only (`<details>`/`:target`/checkbox-hack) либо вынести бургер в демонстрацию открытого состояния отдельным блоком.
- Брейкпоинт **760px → 640px** в `app-home.html`, `leaderboard.html`, `duel-result.html` (норма §4).
- `app-home.html` и все будущие app-страницы: `padding-bottom: calc(56px + max(14px, env(safe-area-inset-bottom)))` под таббар на mobile.
- `duel-result.html`: добавить таббар (mobile) / верхнюю app-навигацию (desktop) — выход с экрана результата должен существовать.
- `duel.html`: добавить пример баннера реконнекта (`.banner--warn`, «восстанавливаем дуэль…») и пример состояний `is-correct`/`is-wrong` у `.opt` (отдельный блок-состояние внизу страницы или дубль экрана).
- `landing.html`: превью-дуэль — ники с модификаторами `--you`/`--rival`.
- `leaderboard.html`: блок `.empty` («в этой теме пока нет игроков») как скрытое/демонстрационное состояние.

### 1.5 Достройка стайлгайда

- `components-base.html`: демо `.modal--paywall` и модалки-формы; статическое focus-состояние полей (класс `.is-focus`-демо); `skeleton--table`.
- PWA: создать `manifest.webmanifest` (name «DiffDuel», short_name, `theme_color: #161B22`, `background_color: #EEF1F5`, иконки-плейсхолдеры 192/512 SVG→PNG-заглушки, `display: standalone`) + `<link rel="manifest">` на все страницы.

## Часть 3. Страницы — весь остаток (§4 мастер-ТЗ)

Требования к составу каждой страницы — в таблицах §4 мастер-ТЗ, они остаются в силе; ниже только уточнения сверх него. Все страницы: адаптив 320–1440, брейкпоинты 880/640, app-страницы с таббаром (mobile), чек-лист §8.

### 3.1 Блок A — публичные

- **A2 `login.html` + `register.html`** — карточка ≤400px, OAuth GitHub (primary) + Google, разделитель `// или по почте`, поля с error-состояниями (показать заполненный и ошибочный варианты), фон — лёгкий VS-сплит. Регистрация: ник с подсказкой доступности, чекбокс оферты.
- **A3 `profile-public.html`** — шапка профиля, `.chart` (динамика Эло), `.tbar--topic` по темам, последние дуэли diff-строками, CTA «Вызвать на дуэль»; вариант «профиль не найден» (блок `.empty` на арена-фоне). Бейджи Founder/Pro в шапке.
- **A5 `tournaments.html`** — карточки турниров: статусы upcoming (`--timer` пилюля + countdown) / live (`--plus`, пульс-точка) / finished (нейтраль); пустое состояние.
- **A6 `founder.html`** — профиль Rifferd (Founder-бейдж), правила пятничного ивента diff-логом, полный зал славы таблицей `.board`.
- **A7 `legal.html`** — шаблон текстовой страницы: контейнер 720px, типографика, оглавление-якоря mono.
- **A8 `error.html` + `offline.html`** — арена-фон, `− 404: страница не найдена` / `− 500` / `− offline` крупным mono, CTA «На главную». Offline — отдельный файл (цель PWA-фолбэка).

### 3.2 Блок B — приложение

- **B2 `queue.html`** — ключевая новая страница, полноэкранная арена (`100dvh`): компонент `.radar`, mono-секундомер поиска (`role="timer"`), индикатор окна «±150 → ±300 → …», кнопка отмены `-dark-ghost`. Три состояния (блоками или якорями на одной странице): поиск / найден (аватары по краям VS-сплита + countdown 3-2-1) / долгий поиск («подключаем призрака», `.ava--ghost`).
- **B5 `training.html`** — карточки тем SQL/JS/Python с `.tbar--topic` прогрессом, сегменты сложности 1–3, CTA.
- **B6 `training-run.html`** — светлая тема, тёмная только код-панель; после ответа: `.opt.is-correct`/`.is-wrong` + блок «Объяснение» (показать оба исхода); прогресс `4/10` mono; внизу — итог сессии (точность, среднее время, слабые темы) отдельным блоком-состоянием.
- **B7 `daily.html`** — задача дня (механика B6) + лидерборд дня (`.board` укороченный) + `.cd` countdown до 23:59.
- **B8 `app-profile.html`** — приватный профиль: всё из A3 + фильтры истории (чипы тем + сегмент период), полный `.chart` 90 дней, серия. Mobile = вкладка таббара «Профиль».
- **B9 `settings.html`** — формы (ник, email, пароль), `.drop` для аватара со всеми состояниями, связанные OAuth-аккаунты (строки с иконками + «отвязать»), опасная зона (бордер `--minus`, confirm-модалка показана открытой в демо-блоке).

### 3.3 Блок C — монетизация

- **C1 `pro.html`** — три `.plan`, таблица сравнения фич (`+`/`−` mono в семантических цветах), FAQ `.acc`.
- **C2 `checkout.html`** — сводка заказа (mono-итог), выбор способа (радио-карточки: ЮKassa / Telegram Stars), кнопка оплаты `-duel`.
- **C3 `billing-result.html`** — два состояния: `+ оплата прошла` (plus) / `− платёж отклонён` (minus) + возврат.
- **C4 `ai-review.html`** — Pro-разбор: по каждой ошибке код-панель, объяснение, diff-блок «− ваш ответ / + верный»; шапка с итогом дуэли.
- **C5 `tournament.html`** — шапка (фонд mono крупно, взнос, `.cd`), таблица участников, CTA входа; состояния live (подсветка текущего раунда) / finished (топ-3 с местами).

### 3.4 Блок D — админка (`admin/`)

Стиль: те же токены, утилитарно — без VS-сплитов и display-шрифта в данных; плотность выше (см. `.adm-table`).

- **D1 `shell.html`** — каркас: `.adm-side` + топбар (поиск, аватар) + контент-зона с заглушкой.
- **D2 `dashboard.html`** — стат-карточки (DAU, дуэли/день, выручка ₽ mono, p95 мс), `.chart`-спарклайны, таблица последних событий.
- **D3 `tasks.html`** — `.adm-table` с фильтрами (тема, статус-пилюли draft/review/published), пагинация, массовые чекбоксы.
- **D4 `task-edit.html`** — форма задачи: селекты, mono-textarea для JSON body, варианты ответов, превью задачи «как в дуэли» (реиспользовать `pre.code` + `.opt`) в правой колонке, кнопки Publish (`-duel`) / Reject (minus-ghost), `.crumbs`.
- **D5 `users.html`** — таблица, поиск, выезжающая карточка пользователя (показать открытой), бан-модалка minus-стиля (открытой в демо).
- **D6 `flags.html`** — список `.flag` с % раскатки.

### 3.5 Обновить `index.html`

Все 22 новые страницы — активными тайлами по блокам A/B/C/D, убрать `tile--soon`, обновить счётчики в hero-мета.

## Часть 4. Приёмка финального прохода

1. Чек-лист §8 мастер-ТЗ — на каждую новую страницу.
2. Глобальный grep `#[0-9A-Fa-f]{3,6}` по всем css/html (вне `tokens.css`, свотчей colors.html, `#fff` на цветных заливках и meta theme-color) — **ноль** вхождений.
3. Ноль `<script>` вне стайлгайда; ноль inline-style с цветами.
4. На каждой странице: `<h1>` (видимый или visually-hidden), `<main>`, корректный таббар/nav по блоку, фиксы 1.3 не регрессированы.
5. `index.html` — полная карта, ни одной битой ссылки.
6. Самопроверка скриншотами: 390px и 1280px для каждой страницы (положить в `scraps/` — каталог уже служебный).

Порядок работы: Часть 1 (фиксы) → Часть 2 (компоненты) → страницы в порядке B2 → A2 → A8 → B5–B9 → A3–A7 → C1–C5 → D1–D6 → Часть 3.5 (индекс) → Часть 4 (приёмка).
